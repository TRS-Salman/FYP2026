Heart Guard: AI-Driven Cardiovascular Diagnostic Assistant1. Project OverviewHeart Guard is an intelligent web-based diagnostic agent designed for the early detection and management of cardiovascular disease (CVD) risks. The application bridges the "information gap" between abstract clinical metrics and patient understanding by providing a user-friendly conversational interface. It utilizes high-performance machine learning (ML) models to analyze 13 key clinical features and provide an immediate, probability-based risk score.2. Key FeaturesConversational Chatbot Interface: Guides users through 13 clinical questions one step at a time to reduce cognitive load and user anxiety.Dual-Algorithm Engine: Employs both Support Vector Machine (SVM) and Multi-Layer Perceptron (MLP) models for reliable risk classification.Real-time Data Preprocessing: Uses a StandardScaler to normalize input features like cholesterol and blood pressure for mathematical accuracy.Secure History Tracking: Features a secure login portal and persistent storage to allow users to monitor their diagnostic history over time.3. Tech StackBackend: Python 3.x Web Framework: Flask (Micro-framework) Machine Learning: Scikit-Learn Model Serialization: Joblib Database: SQLite with SQLAlchemy ORM Frontend: HTML5, CSS3 (Bootstrap 5), and JavaScript (ES6+) 4. System ArchitectureThe project follows a Model-View-Controller (MVC) pattern within a Client-Server Architecture.Frontend (View): Manages conversational states and communicates with the backend via asynchronous JavaScript Fetch API.Backend (Controller): Handles routing, data validation, and connects the UI to the intelligence layer.Intelligence Layer (Model): Pre-trained SVM and MLP models process clinical features to identify risk categories (Healthy vs. Heart Disease).5. Clinical Features AnalyzedThe models evaluate 13 specific parameters established by clinical standards:Age and Sex Chest Pain Type (cp) Resting Blood Pressure (trestbps) Serum Cholesterol (chol) Fasting Blood Sugar (fbs) Resting ECG Results (restecg) Max Heart Rate (thalach) Exercise Induced Angina (exang) ST Depression (oldpeak) Slope of Peak Exercise ST Segment Number of Major Vessels (ca) Thalassemia (thal) 6. Implementation HighlightsAsynchronous Data Collection: JavaScript is used to send data as a POST request to the /predict endpoint without page refreshes.Model Serialization: Trained models are stored as .pkl files and loaded in real-time during inference.Security: Passwords are encrypted using generate_password_hash to meet professional safety standards.7. MethodologyThe development utilized a Hybrid Waterfall-Agile approach. Waterfall planning provided stability for clinical data requirements, while Agile iterations allowed for the continuous refinement of the User Experience (UX) and conversational logic.Author: MD Tayeafur Rahman Salman Supervisor: Yerashenia, Natalia Institution: University of Westminster
